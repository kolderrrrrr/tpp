# Отчет по лабораторной работе 6

## Тема

Разработка WPF-приложения `Анимированный калькулятор` с использованием анимаций.

## Цель работы

Научиться создавать анимации в приложении WPF и реализовать простой калькулятор с анимационными эффектами для улучшения пользовательского интерфейса.

## Краткое описание проекта

В ходе лабораторной работы было создано приложение `Анимированный калькулятор`. Оно выполняет базовые арифметические операции: сложение, вычитание, умножение и деление. Интерфейс содержит дисплей, кнопки цифр от 0 до 9, кнопки операций, кнопку результата `=` и кнопку очистки `C`.

Для улучшения пользовательского опыта добавлены анимации: кнопки увеличиваются при наведении, меняют цвет при нажатии, а результат на дисплее появляется с эффектом затухания.

## Использованные технологии

В работе использовались:

- Microsoft Visual Studio;
- платформа .NET;
- технология Windows Presentation Foundation;
- язык разметки XAML;
- язык программирования C#;
- `Storyboard`;
- `DoubleAnimation`;
- обработчики событий кнопок.

## Ход выполнения работы

Сначала был создан WPF-проект `AnimatedCalculatorWPF`. Главное окно приложения содержит дисплей калькулятора и сетку кнопок.

Фрагмент дисплея:

```xml
<TextBlock x:Name="DisplayTextBlock"
           FontSize="42"
           FontWeight="SemiBold"
           Text="0" />
```

Кнопки калькулятора размещены в сетке `Grid`. Для цифр, операций и специальных действий созданы стили, которые задают внешний вид кнопок.

Фрагмент кнопки:

```xml
<Button Content="7"
        Style="{StaticResource CalculatorButton}"
        Click="NumberButton_Click" />
```

## Функционал калькулятора

Для ввода чисел используется обработчик `NumberButton_Click`. При нажатии на цифру она добавляется на дисплей. Если начинается новый ввод, дисплей очищается и заменяется новым числом.

Фрагмент ввода цифры:

```csharp
private void AppendDigit(string digit)
{
    if (_isNewInput || DisplayTextBlock.Text == "0")
    {
        SetDisplay(digit);
        _isNewInput = false;
        return;
    }

    SetDisplay(DisplayTextBlock.Text + digit);
}
```

Для выполнения операций используется переменная `_storedValue`, в которой хранится первое число, и `_pendingOperation`, в которой хранится выбранная операция. При нажатии `=` выполняется вычисление и результат выводится на дисплей.

Фрагмент вычисления:

```csharp
private bool TryCalculate(double left, double right, string operation, out double result)
{
    result = operation switch
    {
        "+" => left + right,
        "-" => left - right,
        "*" => left * right,
        "/" when Math.Abs(right) > double.Epsilon => left / right,
        _ => double.NaN
    };

    return !double.IsNaN(result) && !double.IsInfinity(result);
}
```

## Анимация кнопок

Для кнопок создан стиль `CalculatorButton`. При наведении курсора запускается `Storyboard` с двумя объектами `DoubleAnimation`, которые изменяют `ScaleX` и `ScaleY`. Благодаря этому кнопка плавно увеличивается. При уходе курсора размер возвращается к исходному.

Фрагмент анимации наведения:

```xml
<DoubleAnimation Storyboard.TargetProperty="RenderTransform.ScaleX"
                 To="1.08"
                 Duration="0:0:0.12" />
<DoubleAnimation Storyboard.TargetProperty="RenderTransform.ScaleY"
                 To="1.08"
                 Duration="0:0:0.12" />
```

При нажатии на кнопку в обработчике `PreviewMouseDown` временно меняется цвет фона, а при отпускании кнопки исходный стиль восстанавливается.

## Анимация дисплея

После выполнения операции результат появляется с эффектом затухания. Для этого в коде создается `Storyboard`, который изменяет прозрачность `DisplayTextBlock` с помощью `DoubleAnimation`.

Фрагмент анимации результата:

```csharp
DoubleAnimation fadeAnimation = new DoubleAnimation
{
    From = 0.2,
    To = 1,
    Duration = TimeSpan.FromMilliseconds(220)
};
```

Для кнопки `C` реализована отдельная анимация очистки дисплея: текст постепенно исчезает, затем отображается значение `0`.

## Обработка ошибок

В калькуляторе предусмотрена обработка деления на ноль. Если пользователь пытается выполнить такую операцию, на дисплее отображается сообщение `Деление на ноль`, а текущее состояние вычислений сбрасывается.
