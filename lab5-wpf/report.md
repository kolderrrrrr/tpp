# Отчет по лабораторной работе 5

## Тема

Разработка WPF-приложения `Блокнот` с использованием команд для управления основными операциями текстового редактора.

## Цель работы

Научиться создавать графический интерфейс в Windows Presentation Foundation и реализовывать команды для создания, открытия и сохранения текстовых документов.

## Краткое описание проекта

В ходе лабораторной работы было создано WPF-приложение `Блокнот`. Приложение содержит многострочную область ввода текста, меню `Файл` и команды для работы с документами.

Пользователь может создать новый документ, открыть существующий текстовый файл, сохранить текущий документ или выбрать новое имя файла при сохранении. Для быстрого доступа используются сочетания клавиш `Ctrl+N`, `Ctrl+O` и `Ctrl+S`.

## Использованные технологии

В работе использовались:

- Microsoft Visual Studio;
- платформа .NET;
- технология Windows Presentation Foundation;
- язык разметки XAML;
- язык программирования C#;
- команды WPF;
- классы `OpenFileDialog` и `SaveFileDialog`;
- файловый ввод-вывод.

## Ход выполнения работы

Сначала был создан WPF-проект `NotepadWPF`. В главном окне приложения была настроена сетка `Grid`, которая содержит меню, текстовую область и строку состояния.

Фрагмент области редактирования:

```xml
<TextBox x:Name="EditorTextBox"
         AcceptsReturn="True"
         AcceptsTab="True"
         TextWrapping="Wrap"
         VerticalScrollBarVisibility="Auto" />
```

Для работы с файлами добавлено меню `Файл`. Пункты меню связаны со стандартными командами WPF.

Фрагмент меню:

```xml
<MenuItem Header="_Новый" Command="New" InputGestureText="Ctrl+N" />
<MenuItem Header="_Открыть..." Command="Open" InputGestureText="Ctrl+O" />
<MenuItem Header="_Сохранить" Command="Save" InputGestureText="Ctrl+S" />
```

## Использование команд

В окне приложения объявлены привязки команд `CommandBinding`. Они связывают команды с обработчиками, которые выполняют нужные действия.

Фрагмент привязки команды:

```xml
<CommandBinding Command="Open"
                Executed="OpenCommand_Executed"
                CanExecute="Command_CanExecute" />
```

Обработчик `CanExecute` разрешает выполнение команд:

```csharp
private void Command_CanExecute(object sender, CanExecuteRoutedEventArgs e)
{
    e.CanExecute = true;
}
```

## Открытие файла

Для открытия файла используется класс `OpenFileDialog`. После выбора файла его содержимое считывается методом `File.ReadAllText` и помещается в текстовую область.

Фрагмент открытия файла:

```csharp
OpenFileDialog dialog = new OpenFileDialog
{
    Filter = "Текстовые файлы (*.txt)|*.txt|Все файлы (*.*)|*.*",
    Title = "Открыть файл"
};

if (dialog.ShowDialog() == true)
{
    EditorTextBox.Text = File.ReadAllText(dialog.FileName, Encoding.UTF8);
}
```

## Сохранение файла

Если документ уже связан с файлом, команда сохранения записывает текущий текст по существующему пути. Если файл еще не выбран, открывается `SaveFileDialog`, где пользователь указывает имя и место сохранения.

Фрагмент сохранения:

```csharp
if (string.IsNullOrWhiteSpace(_currentFilePath))
{
    SaveDocumentAs();
    return;
}

SaveDocument(_currentFilePath);
```

## Обработка ошибок

Операции открытия и сохранения файла помещены в блоки `try-catch`. При возникновении ошибки пользователь получает уведомление через `MessageBox`, а в строке состояния отображается сообщение об ошибке.

Фрагмент обработки ошибки:

```csharp
catch (Exception ex) when (ex is IOException or UnauthorizedAccessException or NotSupportedException)
{
    ShowError("Не удалось открыть файл.", ex);
}
```

## Дополнительные возможности

В приложении отображается имя текущего файла. Если текст был изменен и еще не сохранен, рядом с именем файла и в заголовке окна появляется символ `*`.
