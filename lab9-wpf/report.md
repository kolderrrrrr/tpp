# Отчет по лабораторной работе 9

## Тема

Разработка многоступенчатой формы в WPF с навигацией между страницами.

## Цель работы

Освоить основы навигации между страницами в приложениях WPF, научиться собирать и обрабатывать данные, введенные пользователем на разных этапах.

## Краткое описание проекта

В ходе лабораторной работы было создано WPF-приложение `Многоступенчатая форма регистрации`. Приложение состоит из трех страниц: личные данные, контактные данные и адрес.

Для отображения страниц используется элемент `Frame`, расположенный в главном окне. Каждая страница является отдельным классом, наследуемым от `Page`.

## Использованные технологии

В работе использовались:

- Microsoft Visual Studio;
- платформа .NET;
- технология Windows Presentation Foundation;
- язык разметки XAML;
- язык программирования C#;
- элемент `Frame`;
- классы `Page`;
- `NavigationService`;
- регулярные выражения для проверки email и телефона.

## Структура приложения

Главное окно содержит заголовок и элемент `Frame`, в котором отображается текущая страница формы.

Фрагмент главного окна:

```xml
<Frame x:Name="FormFrame"
       NavigationUIVisibility="Hidden" />
```

При запуске приложение открывает первую страницу:

```csharp
public MainWindow()
{
    InitializeComponent();
    FormFrame.Navigate(new PersonalPage(_formData));
}
```

Для сохранения данных между страницами создан общий класс `FormData`. Один экземпляр этого класса хранится в главном окне и передается во все страницы.

Фрагмент модели:

```csharp
public sealed class FormData
{
    public string FirstName { get; set; } = string.Empty;
    public string LastName { get; set; } = string.Empty;
    public DateTime? BirthDate { get; set; }
    public string Email { get; set; } = string.Empty;
    public string Phone { get; set; } = string.Empty;
}
```

## Страница 1: личные данные

Первая страница содержит поля для имени, фамилии и даты рождения. При нажатии кнопки `Далее` выполняется проверка обязательных полей и корректности даты рождения.

Фрагмент перехода:

```csharp
private void NextButton_Click(object sender, RoutedEventArgs e)
{
    if (!Validate())
    {
        return;
    }

    SaveData();
    NavigationService?.Navigate(new ContactPage(_formData));
}
```

## Страница 2: контактные данные

Вторая страница содержит поля для электронной почты и телефона. Пользователь может вернуться назад или перейти дальше.

Email проверяется регулярным выражением:

```csharp
private static readonly Regex EmailRegex =
    new Regex(@"^[^@\s]+@[^@\s]+\.[^@\s]+$", RegexOptions.Compiled);
```

Телефон проверяется на допустимые символы и длину:

```csharp
private static readonly Regex PhoneRegex =
    new Regex(@"^\+?[0-9\s\-\(\)]{7,20}$", RegexOptions.Compiled);
```

## Страница 3: адрес

Третья страница содержит поля для города, улицы, номера дома и номера квартиры. Номер квартиры является необязательным.

При нажатии кнопки `Отправить` выполняется финальная проверка обязательных полей, после чего все введенные данные отображаются в окне `MessageBox`.

Фрагмент отправки:

```csharp
private void SubmitButton_Click(object sender, RoutedEventArgs e)
{
    if (!Validate())
    {
        return;
    }

    SaveData();
    MessageBox.Show(BuildSummary(), "Введенные данные");
}
```

## Сохранение данных между страницами

Перед каждым переходом текущая страница сохраняет данные в объект `FormData`. При создании страницы данные из этого объекта загружаются обратно в элементы интерфейса.

Так пользователь может переходить назад и вперед без потери введенной информации.

## Валидация

В приложении реализована проверка:

- имя и фамилия не должны быть пустыми;
- дата рождения должна быть выбрана и не должна быть в будущем;
- email должен соответствовать базовому формату;
- телефон должен содержать только допустимые символы и иметь достаточную длину;
- город, улица и номер дома должны быть заполнены.

Если проверка не пройдена, сообщение об ошибке отображается на текущей странице под полями ввода.
