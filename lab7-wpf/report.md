# Отчет по лабораторной работе 7

## Тема

Разработка пользовательского шаблона кнопки в WPF с использованием `ControlTemplate`, `Style`, триггеров и анимаций.

## Цель работы

Научиться разрабатывать и применять пользовательские шаблоны для кнопок в приложениях WPF, освоить принципы кастомизации визуальных элементов управления и работу с визуальными состояниями.

## Краткое описание проекта

В ходе лабораторной работы было создано WPF-приложение `Пользовательский шаблон кнопки`. Приложение демонстрирует несколько кнопок, к которым применяется один общий стиль с пользовательским `ControlTemplate`.

Шаблон изменяет стандартный внешний вид кнопки: добавляет фон с рамкой, верхний световой слой, центрированный контент и анимированное масштабирование при наведении. Также реализованы отдельные состояния для наведения, нажатия и отключенной кнопки.

## Наблюдения по стандартной кнопке

Стандартный элемент `Button` в WPF поддерживает свойства `Background`, `Foreground`, `BorderBrush`, `FontSize`, `Padding`, `IsMouseOver`, `IsPressed` и `IsEnabled`. Однако его внешний вид зависит от стандартной темы Windows, поэтому для полного контроля над оформлением удобнее использовать собственный `ControlTemplate`.

В пользовательском шаблоне можно явно задать структуру кнопки и определить, как она будет выглядеть в каждом состоянии.

## Проектирование ControlTemplate

Основой шаблона стал элемент `Border`, который отвечает за фон, рамку и скругление углов. Внутри него расположен `Grid`, содержащий декоративный верхний слой `TopGlow` и `ContentPresenter` для отображения текста кнопки.

Фрагмент шаблона:

```xml
<ControlTemplate TargetType="Button">
    <Border x:Name="RootBorder"
            Background="{TemplateBinding Background}"
            BorderBrush="{TemplateBinding BorderBrush}"
            BorderThickness="{TemplateBinding BorderThickness}">
        <Grid>
            <Border x:Name="TopGlow" />
            <ContentPresenter HorizontalAlignment="Center"
                              VerticalAlignment="Center" />
        </Grid>
    </Border>
</ControlTemplate>
```

Для передачи значений из стиля внутрь шаблона используются привязки `TemplateBinding`. Это позволяет менять `Background`, `Foreground`, `BorderBrush` и `Padding` у кнопки, не переписывая шаблон.

## Визуальные состояния

В шаблоне реализованы три состояния:

- `MouseOver` - кнопка меняет фон, световой слой становится заметнее, а размер плавно увеличивается;
- `Pressed` - фон становится темнее, кнопка слегка уменьшается, создавая эффект нажатия;
- `Disabled` - кнопка получает серый фон, пониженную прозрачность и недоступный вид.

Фрагмент состояния наведения:

```xml
<Trigger Property="IsMouseOver" Value="True">
    <Setter TargetName="RootBorder"
            Property="Background"
            Value="{StaticResource HoverBackgroundBrush}" />
</Trigger>
```

## Анимации

Для плавного изменения размера при наведении используется `Storyboard` с двумя объектами `DoubleAnimation`. Они изменяют свойства `ScaleX` и `ScaleY` у `ScaleTransform`.

Фрагмент анимации:

```xml
<DoubleAnimation Storyboard.TargetName="RootBorder"
                 Storyboard.TargetProperty="(UIElement.RenderTransform).(ScaleTransform.ScaleX)"
                 To="1.04"
                 Duration="0:0:0.15" />
```

При уходе курсора запускается обратная анимация, которая возвращает кнопку к исходному размеру.

## Применение стиля

Стиль `TemplateButtonStyle` размещен в ресурсах окна и применен к нескольким кнопкам. Для одной кнопки создан вариант `AccentButtonStyle`, который наследует основной стиль, но задает другой цвет фона и рамки.

Фрагмент применения:

```xml
<Button Content="Сохранить"
        Style="{StaticResource TemplateButtonStyle}"
        Click="ActionButton_Click" />
```

## Тестирование

В приложении проверены основные состояния кнопки: обычное состояние, наведение курсора, нажатие и отключение. Кнопки сохраняют стандартную функциональность: при нажатии выполняется обработчик события, а запись о действии добавляется в журнал.

Также проверено, что шаблон корректно отображает разный текст кнопок и может использоваться повторно для нескольких элементов.
