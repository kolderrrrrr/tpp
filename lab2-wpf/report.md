# Отчет по лабораторной работе 2

## Тема

Разработка графического приложения-калькулятора с использованием Windows Presentation Foundation.

## Цель работы

Научиться разрабатывать графические приложения WPF, познакомиться с элементами пользовательского интерфейса, обработкой событий и реализовать основные арифметические операции.

## Краткое описание проекта

В ходе лабораторной работы было создано WPF-приложение `Калькулятор`. Приложение позволяет вводить арифметические выражения с помощью кнопок, отображает введенные данные в текстовом поле и вычисляет результат после нажатия кнопки `=`.

Калькулятор поддерживает:

- цифры от `0` до `9`;
- операции `+`, `-`, `*`, `/`;
- десятичные дроби;
- скобки;
- очистку поля ввода;
- историю вычислений;
- обработку ошибок при некорректном вводе и делении на ноль.

## Использованные технологии

В работе использовались:

- Microsoft Visual Studio;
- платформа .NET;
- технология Windows Presentation Foundation;
- язык разметки XAML;
- язык программирования C#.

Интерфейс приложения описан в файле `MainWindow.xaml`, а обработка нажатий кнопок и вычисление выражений реализованы в файле `MainWindow.xaml.cs`.

## Ход выполнения работы

Сначала был создан проект WPF-приложения. В главном окне были размещены текстовое поле для отображения выражения и результата, кнопки цифр и арифметических операций, кнопки очистки и вычисления результата, а также список для истории вычислений.

Фрагмент разметки кнопки:

```xml
<Button Content="=" Grid.Row="4" Grid.Column="3"
        Tag="Equals"
        Click="EqualsButton_Click"
        Style="{StaticResource EqualsButton}" />
```

При нажатии на кнопку цифры или операции значение из свойства `Tag` добавляется в текстовое поле. При нажатии кнопки `C` поле очищается. При нажатии кнопки `=` выражение передается в класс `ExpressionParser`, который вычисляет результат.

Фрагмент обработчика вычисления:

```csharp
private void EqualsButton_Click(object sender, RoutedEventArgs e)
{
    string expression = DisplayTextBox.Text;

    try
    {
        double result = new ExpressionParser(expression).Parse();
        string resultText = result.ToString("G10", CultureInfo.InvariantCulture);

        DisplayTextBox.Text = resultText;
        HistoryListBox.Items.Insert(0, expression + " = " + resultText);
        _isResultShown = true;
    }
    catch (Exception ex)
    {
        MessageBox.Show(ex.Message, "Ошибка вычисления", MessageBoxButton.OK, MessageBoxImage.Warning);
    }
}
```

Для правильного порядка операций была реализована последовательная обработка выражения: сначала числа и скобки, затем умножение и деление, затем сложение и вычитание. Благодаря этому выражение `2+3*4` дает результат `14`, а выражение `(2+3)*4` дает результат `20`.

## Обработка ошибок

В приложении предусмотрена обработка некорректного ввода. Если выражение записано неправильно, пользователю показывается сообщение об ошибке. Также реализована отдельная проверка деления на ноль.

Фрагмент проверки:

```csharp
if (Math.Abs(nextValue) < 0.000000001)
{
    throw new DivideByZeroException("Деление на ноль невозможно.");
}
```

## Возникшие трудности

Основная сложность заключалась в реализации вычисления выражений с учетом приоритета операций и скобок. Для решения этой задачи был создан отдельный класс `ExpressionParser`, который разбирает выражение по частям и выполняет операции в правильном порядке.

## Вывод

В результате выполнения лабораторной работы было создано графическое WPF-приложение-калькулятор. Были изучены принципы построения интерфейса на XAML, обработка событий кнопок в C#, работа с текстовым полем, вывод сообщений об ошибках и реализация логики арифметических вычислений.
