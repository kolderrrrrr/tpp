# Отчет по лабораторной работе 4

## Тема

Разработка WPF-приложения с использованием механизма привязки данных на примере игры `Угадай число`.

## Цель работы

Научиться создавать WPF-приложение с привязкой данных, моделью состояния, обработкой пользовательского ввода и обновлением интерфейса через интерфейс `INotifyPropertyChanged`.

## Краткое описание проекта

В ходе лабораторной работы было создано приложение `Угадай число`. При запуске программа генерирует случайное число в диапазоне от 1 до 100. Пользователь вводит предполагаемое число, нажимает кнопку проверки и получает подсказку: введенное значение больше или меньше загаданного числа.

Приложение отображает количество использованных попыток и позволяет начать новую игру без закрытия окна.

## Использованные технологии

В работе использовались:

- Microsoft Visual Studio;
- платформа .NET;
- технология Windows Presentation Foundation;
- язык разметки XAML;
- язык программирования C#;
- механизм привязки данных;
- интерфейс `INotifyPropertyChanged`.

## Ход выполнения работы

Сначала был создан WPF-проект `GuessNumberWPF`. В главном окне приложения размещены элементы управления: поле ввода `TextBox`, кнопка проверки, текстовые блоки для подсказки, результата и количества попыток, а также кнопка запуска новой игры.

Фрагмент привязки текста подсказки:

```xml
<TextBlock Text="{Binding Hint}" />
```

Для хранения состояния игры создан класс `GameState`. В нем находятся загаданное число, текущий ввод пользователя, количество попыток и сообщение для отображения результата.

Фрагмент модели:

```csharp
public sealed class GameState : INotifyPropertyChanged
{
    private const int MinNumber = 1;
    private const int MaxNumber = 100;
    private int _targetNumber;
    private int _attempts;

    public event PropertyChangedEventHandler? PropertyChanged;
}
```

В конструкторе главного окна объект `GameState` назначается контекстом данных:

```csharp
public MainWindow()
{
    InitializeComponent();
    DataContext = _gameState;
}
```

## Привязка данных

Для автоматического обновления интерфейса класс `GameState` реализует интерфейс `INotifyPropertyChanged`. При изменении свойства вызывается метод `OnPropertyChanged`, после чего WPF обновляет связанные элементы интерфейса.

Фрагмент уведомления об изменении свойства:

```csharp
private void OnPropertyChanged([CallerMemberName] string? propertyName = null)
{
    PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
}
```

Количество попыток выводится через вычисляемое свойство `AttemptsText`, которое обновляется при каждом изменении счетчика.

## Обработка ввода

Перед проверкой введенного значения программа убеждается, что пользователь ввел целое число. Также выполняется проверка диапазона от 1 до 100. Если ввод некорректный, пользователю выводится сообщение об ошибке, а количество попыток не увеличивается.

Фрагмент проверки:

```csharp
if (!int.TryParse(CurrentGuess.Trim(), out int guess))
{
    SetMessage("Введите целое число.", Brushes.Red);
    return;
}

if (guess < MinNumber || guess > MaxNumber)
{
    SetMessage($"Число должно быть в диапазоне от {MinNumber} до {MaxNumber}.", Brushes.Red);
    return;
}
```

Если число меньше загаданного, выводится сообщение `Слишком маленькое число`. Если больше, выводится сообщение `Слишком большое число`. При совпадении отображается поздравление и итоговое количество попыток.

## Дополнительные возможности

В приложении реализована кнопка `Новая игра`. Она генерирует новое случайное число, сбрасывает количество попыток, очищает поле ввода и обновляет сообщение состояния.
